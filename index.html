ระบบ Visitor Management
ใครทำอะไร อยู่ตรงไหนของระบบ

ทำความเข้าใจระบบบริหารผู้มาเยือนของหมู่บ้าน ตั้งแต่ผู้ดูแลระบบสูงสุด ไปจนถึง รปภ. ที่หน้าป้อม ซึ่งเป็นหัวใจของการใช้งานจริง — คลิกสำรวจได้ทุกบทบาท

02 · ผังรวม

โครงสร้างทั้งระบบ — คลิกเพื่อสำรวจ

คลิกที่กล่องใดก็ได้เพื่อดูรายละเอียด ระบบจะไฮไลต์ “เส้นทาง” จากผู้ดูแลสูงสุดลงมา และถ้าคลิกที่ แอดมินที่เป็นคนเดียวกัน (เช่น BOOK) จะเห็นว่าเขาดูแลกี่สาขา — กดหัวลูกศรเพื่อย่อ/ขยายกิ่ง

คน · 1 คนมีได้หลาย role สถานที่ · หมู่บ้าน / สาขา / ป้อม อุปกรณ์ · LCR & KIOSK / Handheld จุดเน้น · ผู้ใช้งาน App ยาม
เน้นตามแกน:

กล่องที่มีกรอบสีทอง = รปภ ประจำป้อม ผู้ใช้งาน App ยาม โดยตรง · ป้อมถูกพับไว้ กดขยายดูได้

    เลือกดูรายละเอียด
    คลิกที่กล่องใด ๆ ในผัง เพื่อดูว่าบทบาทนั้นอยู่ที่ไหน ทำหน้าที่อะไร และใช้อุปกรณ์อะไรบ้าง
    03 · ขั้นตอนการตั้งค่า

    ตั้งค่าระบบทีละสเต็ป — จาก Admin สูงสุด สู่ App ยาม

    ไล่ตั้งแต่ผู้ดูแลระบบทั้งหมด → เพิ่มหมู่บ้าน/สาขา → เพิ่ม Admin ประจำสาขา → สร้าง Role และรับ Code → Sync Action → จนถึงหน้า App ของ รปภ ที่หน้างานจริง โครงสร้างเดียวกับโปรเจกต์เดิม DGA ของคุณ

    1
    เทียบกับ DGA

    Admin ผู้ดูแลระบบทั้งหมด = เหมือนโปรเจกต์ DGA เดิม

    ชั้นบนสุดคือ Admin ดูแลระบบทั้งหมด มองเห็นและจัดการได้ทุก Partner ทุกหมู่บ้าน — แนวคิดเดียวกับหน้า “จัดการ DGA Partner” ในโปรเจกต์เดิมของคุณเป๊ะ ๆ

    • ✓เห็น ทุก Partner / หมู่บ้าน ในระบบจากที่เดียว
    • ✓กด “+ เพิ่ม Partner” เพื่อเริ่มสร้างหมู่บ้านใหม่ (ไปต่อสเต็ป 2)
    Admin ดูแลระบบทั้งหมด — มุมมองเดียวกับ DGA Partner
    🖼️
    วางสกรีนช็อตหน้า Admin/DGA ที่ images/1.png
    2
    เพิ่ม Partner

    เพิ่มหมู่บ้าน + สาขา (และ Group ของแต่ละที่)

    หลังกด “เพิ่ม Partner” จะเป็นการสร้าง หมู่บ้าน และ สาขา โดยแต่ละเครือมี Group ครอบอยู่ เช่น LAND AND HOUSE เป็น Group ด้านนอก แล้วแตกออกเป็นสาขาย่อยตามทำเล

    🏢 LAND AND HOUSE → ภูเก็ต กรุงเทพ เชียงใหม่
    • ✓Group ด้านนอก = เครือหมู่บ้าน (เช่น LAND AND HOUSE)
    • ✓ภายในแตกเป็น หลายสาขา ตามทำเล (ภูเก็ต / กรุงเทพ / เชียงใหม่)
    รายการ Partner — LAND AND HOUSE แตกเป็นหลายสาขา
    🖼️
    วางสกรีนช็อตรายการหมู่บ้าน/สาขา ที่ images/2.png
    2.5
    เพิ่ม Admin สาขา

    เพิ่ม Admin ผู้จัดการ “สาขา 1 ภูเก็ต”

    กำหนด Admin ประจำสาขา ขึ้นมาดูแลเฉพาะสาขานั้น ๆ — ระบุชื่อผู้ดูแล เบอร์ติดต่อ และสถานะการใช้งาน (Enabled / Disabled)

    • ✓เพิ่ม Admin ผูกกับ สาขา 1 ภูเก็ต โดยตรง
    • ✓คุมสถานะ Enabled / Disabled / Deleted ของผู้ดูแลแต่ละคน
    เพิ่ม Admin จัดการสาขา 1 ภูเก็ต
    🖼️
    วางสกรีนช็อตหน้าเพิ่ม Admin สาขา ที่ images/3.png
    3
    สร้าง Role + รับ Code

    สร้าง Role ของสาขา — ใครทำหน้าที่อะไร ประจำที่ไหน

    ในแต่ละสาขากำหนด Role (เช่น รปภ ประจำป้อม, หัวหน้าชุด รปภ) คู่กับ ตำแหน่งที่ประจำการ (ป้อมประตูหน้า / ประตูหลัง) เมื่อสร้างเสร็จระบบจะออก Code มาให้

    • 🎖️ROLE — รปภ ประจำป้อม, หัวหน้าชุด รปภ ฯลฯ
    • 📍WHERE — ประจำการที่ ป้อมประตูหน้า / ป้อมประตูหลัง
    🔑 สร้าง Role เสร็จ ระบบออก Code ให้ — เอา Code นี้ไปกรอกในหน้า App ของ User (รปภ) GC-FPA000-2375B
    Role × ตำแหน่งประจำการ ของสาขา 1 ภูเก็ต พร้อม Code/Share
    🖼️
    วางสกรีนช็อตหน้า Role + Code ที่ images/4.png
    4
    Sync Action

    เอา Group จากสเต็ป 3 ไป Sync กับ Action

    นำ Group (Role) ที่สร้างไว้ มาจับคู่กับ Action ที่อนุญาตให้ทำได้ — เลือกว่า Group นี้ทำอะไรได้บ้าง เช่น เปิด/ปิดไม้กั้น, สมัครสมาชิกให้ Visitor, ดู Dashboard, ดู LOG

    • ✓เปิด / ปิด ไม้กั้น
    • ✓สมัครสมาชิกให้ Visitor
    • ✓ดู Dashboard · ดู LOG
    เอา Group ไป Sync กับ Action (Services)
    🖼️
    วางสกรีนช็อตหน้า Sync Action ที่ images/5.png
    5
    หน้า App รปภ

    App ของยาม — ขึ้นเฉพาะ Action ที่ถูก Sync

    ฝั่งหน้า App ของ รปภ จะแสดงเฉพาะ Action ที่ถูก Sync มาให้ Group นั้น เท่านั้น เปรียบเหมือนการ “ดาวน์โหลดแอป” จาก App Store — มีสิทธิ์อันไหน อันนั้นก็โผล่ให้กดใช้
