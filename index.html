Visitor · อุปกรณ์ที่แต่ละ Role แตะ

ใครเจออุปกรณ์อะไร — ไล่ตาม Check Point

เลือกกลุ่มผู้มาเยือนด้านล่าง ระบบจะไล่ให้ดูว่าตั้งแต่ ก่อนมาถึง → เข้า → ระหว่างอยู่ → ออก เขาต้อง interact กับ อุปกรณ์ ตัวไหนบ้าง — สีของชิปบอกว่าอุปกรณ์อยู่โซนไหน

มือถือผู้มาเยือน · Line OA · Beacon · Bio · Web/App จุดบริการ/ป้อม · Kiosk · Render · อุปกรณ์ รปภ ช่องทางรถ · LPR · ไม้กั้น · QR จ่ายเงิน
  • ✓ต่างกันแค่ 3 แกน — Walk-in vs จองล่วงหน้า · EV (จ่ายเงิน) vs ไม่ EV · Visitor (ครั้งเดียว) vs Internal/User (ใช้ซ้ำ)
  • ✓จองล่วงหน้า = แตะอุปกรณ์น้อยลง — ผ่าน Beacon + LPR อัตโนมัติ ไม่ต้องลงทะเบียนที่ป้อม
  • ✓Handheld กับ Tablet ทำงานเดียวกัน — รปภ ใช้ verify ข้อมูลจาก LPR จึงรวมเป็น "อุปกรณ์ รปภ" ตัวเดียว

โครงสร้างทั้งระบบ — คลิกเพื่อสำรวจ

คลิกที่กล่องใดก็ได้เพื่อดูรายละเอียด ระบบจะไฮไลต์ "เส้นทาง" จากผู้ดูแลสูงสุดลงมา — กดหัวลูกศรเพื่อย่อ/ขยายกิ่ง

    เทียบกับ DGA

    Admin ผู้ดูแลระบบทั้งหมด

    ชั้นบนสุดคือ Admin ดูแลระบบทั้งหมด มองเห็นและจัดการได้ทุก Partner ทุกหมู่บ้าน — แนวคิดเดียวกับหน้า "จัดการ DGA Partner" ในโปรเจกต์เดิมเป๊ะ ๆ

    • ✓เห็น ทุก Partner / หมู่บ้าน ในระบบจากที่เดียว
    • ✓กด "+ เพิ่ม Partner" เพื่อเริ่มสร้างหมู่บ้านใหม่ (ไปต่อสเต็ป 2)
    Admin ดูแลระบบทั้งหมด — มุมมองเดียวกับ DGA Partner
    2
    เพิ่ม Partner

    เพิ่มหมู่บ้าน + สาขา — ออก Code ประจำกลุ่ม

    เมื่อสร้างกลุ่ม (เครือหมู่บ้าน / สาขา) ลองสมมุติ Code ประจำกลุ่มขึ้นมาสักอัน แล้วแนบChecksumที่คำนวณจาก Code นั้นไปด้วย — Checksum คือตัวสั้น ๆ ที่ได้จากสูตรเดียวกันทั้งหน้าบ้าน (App)และหลังบ้าน (Server) ใช้เช็คว่าทั้งสองฝั่งถือ Code ตรงกันไหม

    LAND AND HOUSE → ภูเก็ต กรุงเทพ เชียงใหม่
    สมมุติ Code ของกลุ่ม แล้วแนบ Checksum ที่คำนวณจาก Code นั้นมาด้วย GRP-LH-0001 ✓ sum 7F3A
    • ✓Code ประจำกลุ่ม — ตัวระบุกลุ่ม เช่น GRP-LH-0001
    • ✓Checksum — คำนวณจาก Code ด้วยสูตรเดียวกันทั้งหน้าบ้านและหลังบ้าน
    • ✓ใช้เทียบว่า Code ที่หน้าบ้านถืออยู่ ตรงกับที่หลังบ้านออกให้ — ไม่เพี้ยน ไม่ถูกแก้
    แต่ละกลุ่มมี Code ประจำตัว (+ Checksum) สำหรับตรวจสอบ
    2.5
    เพิ่ม Admin สาขา

    เพิ่ม Admin จัดการ "สาขา 1 ภูเก็ต"

    กำหนด Admin ประจำสาขา ขึ้นมาดูแลเฉพาะสาขานั้น ๆ — ระบุชื่อผู้ดูแล เบอร์ติดต่อ และสถานะการใช้งาน (Enabled / Disabled)

    • ✓เพิ่ม Admin ผูกกับ สาขา 1 ภูเก็ต โดยตรง
    • ✓คุมสถานะ Enabled / Disabled / Deleted ของผู้ดูแลแต่ละคน
    เพิ่ม Admin จัดการสาขา 1 ภูเก็ต
    3
    สร้าง Role + รับ Code

    สร้าง Role ของสาขา — ใครทำอะไร ประจำที่ไหน

    ในแต่ละสาขากำหนด Role (เช่น รปภ ประจำป้อม, หัวหน้าชุด รปภ) คู่กับ ตำแหน่งที่ประจำการ (ป้อมประตูหน้า / ประตูหลัง) เมื่อสร้างเสร็จระบบจะ gen Code มาให้ พร้อม Checksum สำหรับตรวจสอบความถูกต้อง — เหมือนกับ Code ของกลุ่มในสเต็ปก่อนหน้า

    • ✓ROLE — รปภ ประจำป้อม, หัวหน้าชุด รปภ ฯลฯ
    • ✓WHERE — ประจำการที่ ป้อมประตูหน้า / ป้อมประตูหลัง
    • ✓Code + Checksum — ทุก Code ที่ระบบ gen ออกมา จะมี Checksum กำกับเสมอ
    สร้าง Role เสร็จ ระบบ gen Code + Checksum ให้ — เอา Code นี้ไปกรอกในหน้า App ของ User (รปภ) GC-FPA000-2375B ✓ sum 9C2D
    Role × ตำแหน่งประจำการ ของสาขา 1 ภูเก็ต พร้อม Code/Share
    3.5
    ตรวจ Code ด้วย Checksum

    เอา Code ไปเช็ค — Checksum ช่วยให้ไม่ต้องยิง API ตรงนี้

    พอหน้าบ้าน (App) รับ Code ที่กรอกเข้ามา ก็คำนวณ Checksum เองในเครื่องด้วยสูตรเดียวกับหลังบ้าน แล้วเทียบกับ Checksum ที่แนบมากับ Code — ถ้าตรง ก็รู้ทันทีว่า Code ฟอร์แมตถูก ไม่พิมพ์ผิด โดยไม่ต้องยิง API ไปถาม Server ตรงจุดนี้

    หน้าบ้านคำนวณ Checksum จาก Code ที่กรอก แล้วเทียบกับตัวที่แนบมา — ตรง = ผ่าน GC-FPA000-2375B ✓ sum 9C2D ตรง
    • ✓เร็ว — เช็คเสร็จในเครื่อง ไม่ต้องรอ network round-trip
    • ✓ลดภาระ Server — ไม่ต้องยิง API ทุกครั้งที่ User กรอก Code
    • ✓กันพิมพ์ผิด — Code เพี้ยนนิดเดียว Checksum ก็ไม่ตรง รู้ทันทีก่อนส่ง
    หน้าบ้านเทียบ Checksum เองก่อน — ไม่ต้องยิง API ตรงจุดนี้
    หมายเหตุ: Checksum ยืนยันแค่ว่า Code ฟอร์แมตถูก / ไม่เพี้ยน เท่านั้น — ส่วนจะเป็น Code ที่มีจริงและมีสิทธิ์ไหม ค่อยให้หลังบ้านยืนยันตอนเอาไปใช้งานจริง
    4
    Sync Action

    เอา Group จากสเต็ป 3 ไป Sync กับ Action

    นำ Group (Role) ที่สร้างไว้ มาจับคู่กับ Action ที่อนุญาตให้ทำได้ — โดยแต่ละ Action ที่ Sync จะมีCode ประจำ ActionและChecksumกำกับ เพื่อให้ฝั่ง App เทียบ Checksum แล้วรู้ว่าสิทธิ์ที่ได้รับมา ตรงและไม่ถูกแก้

    • ✓เปิด / ปิด ไม้กั้น
    • ✓สมัครสมาชิกให้ Visitor
    • ✓ดู Dashboard · ดู LOG
    แต่ละ Action ที่ Sync มี Code + Checksum ของตัวเอง — App เทียบ Checksum ก่อนเปิดให้ใช้ ACT-GATE-01 ✓ sum B1E0
    เอา Group ไป Sync กับ Action (Services)

    หน้า App รปภ

    App ของยาม — ขึ้นเฉพาะ Action ที่ถูก Sync

    ฝั่งหน้า App ของ รปภ จะแสดงเฉพาะ Action ที่ถูก Sync มาให้ Group นั้น เท่านั้น เปรียบเหมือนการ "ดาวน์โหลดแอป" จาก App Store — มีสิทธิ์อันไหน อันนั้นก็โผล่ให้กดใช้
